# Auth Service 系统梳理文档

## 1. 项目概述

`auth-service` 是基于 **Spring Boot + Spring Cloud + DDD（领域驱动设计）** 构建的认证微服务，负责用户注册、登录及 JWT Token 签发/验证。

- **服务名**：`auth-service`
- **默认端口**：`8080`
- **数据库**：开发环境 H2 内存库，生产环境 MySQL（`prod` profile）
- **注册中心**：Nacos（默认关闭，生产环境启用）

---

## 2. 模块分层（DDD 四层架构）

```
com.example.auth

    └── security/       ─ JWT、Spring Security
```

## 3. 核心领域模型

### 3.1 用户聚合根 `User`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `UserId`（值对象） | UUID，不可变 |
| `username` | `String` | 长度 3–32 |
| `email` | `Email`（值对象） | 格式校验 |
| `password` | `Password`（值对象） | BCrypt 加密存储 |
| `status` | `UserStatus` | `ACTIVE` / `INACTIVE` / `LOCKED` |
| `createdAt` | `LocalDateTime` | 不可变 |
| `updatedAt` | `LocalDateTime` | 每次变更更新 |

**业务行为**

| 方法 | 描述 |
|------|------|
| `User.register(...)` | 工厂方法，创建新用户，触发 `UserRegisteredEvent` |
| `user.login(...)` | 校验密码、账号状态，触发 `UserLoggedInEvent` |
| `user.lock()` | 锁定账号 |
| `user.activate()` | 激活账号 |
| `user.pullDomainEvents()` | 拉取并清空待发布的领域事件 |

**创建规则（注册校验）**

- `username`：非空，长度 3–32
- `password`：非空，最少 6 位
- `email`：格式合法（由 `Email` 值对象保证）
- 邮箱、用户名唯一性由 `UserDomainService` 在应用层调用前校验

### 3.2 值对象

| 类 | 说明 |
|----|------|
| `UserId` | UUID 封装，`generate()` 生成新 ID，`of(id)` 重建 |
| `Email` | 正则格式校验，不可变 |
| `Password` | 存储编码值，`matches()` 委托 `PasswordEncoder` 比对 |

### 3.3 领域事件

| 事件 | 触发时机 | 携带字段 |
|------|----------|---------|
| `UserRegisteredEvent` | 注册成功 | `userId`, `username`, `email` |
| `UserLoggedInEvent` | 登录成功 | `userId`, `username` |

事件通过 Spring `ApplicationEventPublisher` 发布（应用层统一发布）。

## 4. 应用层用例

### `AuthApplicationService`

| 方法 | 用例 | 流程 |
|------|------|------|
| `register(RegisterCommand)` | 用户注册 | 唯一性校验 → 创建聚合 → 持久化 → 发布事件 |
| `login(LoginCommand)` | 用户登录 | 查询用户 → 聚合登录校验 → 持久化 → 签发 Token → 发布事件 |
| `getUserById(String)` | 查询用户信息 | 根据 ID 查询，返回 `UserInfo` |

## 5. API 接口

### 5.1 认证接口（`/api/auth/**`，无需认证）

#### 注册
```
POST /api/auth/register
Content-Type: application/json

{
  "username": "alice",
  "email": "alice@example.com",
  "password": "123456"
}
```
- 成功：`201 Created`，`{ "code": 200 }`
- 失败：`400 Bad Request`，`{ "code": 400, "message": "..." }`

#### 登录
```
POST /api/auth/login
Content-Type: application/json

{
  "email": "alice@example.com",
  "password": "123456"
}
```
- 成功：`200 OK`
```json
{
  "code": 200,
  "data": {
    "token": "<JWT>",
    "userId": "...",
    "username": "alice",
    "email": "alice@example.com"
  }
}
```

#### 验证当前用户
```
GET /api/auth/me
Authorization: Bearer <token>
```
- 成功：`200 OK`，`{ "code": 200, "data": "Authenticated, userId: ..." }`
- 未认证：`401 Unauthorized`，`{ "code": 401, "message": "Unauthorized" }`

### 5.2 用户接口（`/api/user/**`，需要 JWT 认证）

#### 欢迎页信息
```
GET /api/user/welcome
Authorization: Bearer <token>
```
- 成功：`200 OK`
```json
{
  "code": 200,
  "data": {
    "userId": "...",
    "username": "alice",
    "email": "alice@example.com",
    "createdAt": "2026-03-25T10:00:00"
  }
}
```

## 6. 认证流程

```
客户端                          服务端
  │                               │
  │  POST /api/auth/login         │
  │──────────────────────────────►│
  │                               │  UserRepository.findByEmail()
  │                               │  User.login(password, encoder)
  │                               │  JwtTokenProvider.generateToken()
  │  { token: "eyJ..." }          │
  │◄──────────────────────────────│
  │                               │
  │  GET /api/user/welcome        │
  │  Authorization: Bearer eyJ... │
  │──────────────────────────────►│
  │                               │  JwtAuthenticationFilter
  │                               │  → tokenService.validateToken()
  │                               │  → 注入 SecurityContext
  │  { username, email, ... }     │
  │◄──────────────────────────────│
```

### JWT 过滤器处理链

```
请求 → JwtAuthenticationFilter
         ├── 提取 Authorization: Bearer <token>
         ├── tokenService.validateToken(token)
         ├── 解析 userId / username
         └── 写入 SecurityContextHolder
```

## 7. 基础设施层

### 7.1 持久化

| 类 | 职责 |
|----|------|
| `UserRepositoryImpl` | 实现领域 `UserRepository` 接口，负责 Domain ↔ PO 转换 |
| `UserJpaRepository` | Spring Data JPA 接口，提供基础 CRUD |
| `UserPO` | JPA 持久化对象，映射 `t_user` 表 |

**领域对象与 PO 转换规则**
- `User → UserPO`：`toUserPO()`，直接字段映射
- `UserPO → User`：`User.reconstitute()`，不触发领域事件

### 7.2 安全配置

| 放行路径 | 说明 |
|----------|------|
| `/api/auth/**` | 注册、登录接口 |
| `/h2-console/**` | 开发调试 |
| `/actuator/health` | 健康检查 |
| `/*.html` | 静态页面（login.html 等） |
| 其他所有路径 | 需要 JWT 认证 |

Session 策略：`STATELESS`（无状态，不创建 HttpSession）

### 7.3 JWT 配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| `jwt.secret` | `auth-service-jwt-secret-key-minimum-32chars!` | HS256，≥32 字符 |
| `jwt.expiration` | `86400000` ms | 有效期 24 小时 |

## 8. 配置与环境

### 开发环境（默认）
- 数据库：H2 内存库，`ddl-auto: create-drop`
- H2 控制台：`http://localhost:8080/h2-console`（JDBC URL: `jdbc:h2:mem:authdb`）
- Nacos：关闭

### 生产环境（`--spring.profiles.active=prod`）
- 数据库：MySQL，`ddl-auto: validate`
- 环境变量：`DB_USERNAME`、`DB_PASSWORD`
- Nacos 注册中心 & 配置中心：启用（`server-addr: 127.0.0.1:8848`）
